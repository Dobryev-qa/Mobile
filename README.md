## Project Overview

**Homebody Challenge** is an iOS mobile fitness application designed for home-based workouts.  
The app allows users to join monthly challenges, create custom workouts, track progress, and receive workout reminders via notifications.

This repository demonstrates **manual QA work** performed on the mobile application.

- Platform: iOS
- App type: Offline-first mobile application
- Technology: Flutter (Dart)
- Role: Manual QA Engineer
- Testing focus: Functional, Smoke, Regression, Exploratory testing

## QA Documentation

All testing artifacts are structured and linked below:

- **[User Stories](https://github.com/Dobryev-qa/Mobile/blob/main/User_Stories.md)**  
  User requirements describing what the app should do from the user point of view.

- **[Acceptance Criteria](https://github.com/Dobryev-qa/Mobile/blob/main/Accept_Criteria.md)**  
  Rules that explain when a user story is completed successfully.

- **[Test Plan](https://github.com/Dobryev-qa/Mobile/blob/main/Test_Plan.md)**  
  Scope, objectives, test types, environment, risks, and exit criteria.

- **[Checklists](https://github.com/Dobryev-qa/Mobile/blob/main/%D0%A1hecklist.md)**  
  Smoke and functional checklists for fast validation.

- **[Test Cases](https://github.com/Dobryev-qa/Mobile/blob/main/Test_Cases.md)**  
  Manual test cases for main app features.

- **[Exploratory Testing](https://github.com/Dobryev-qa/Mobile/blob/main/Exploratory_Testing.md)**
